# RecipeWizard

Приложение-помощник по рецептам на C++ и Qt 6 (QML). Хранит коллекцию
рецептов в базе SQLite, подбирает блюда под имеющиеся продукты, фильтрует по тегам
и калорийности и ведёт дневник питания.

---

## Назначение

RecipeWizard решает простую бытовую задачу: «что приготовить из того, что есть дома».
Пользователь отмечает продукты в своём виртуальном холодильнике, а программа предлагает
рецепты, ранжируя их по числу совпавших ингредиентов - сначала те, для которых почти
ничего докупать не нужно. Дополнительно приложение помогает следить за калориями и БЖУ.

## Функционал

- **Каталог рецептов.** База из 55 готовых рецептов с фото, тегами, калорийностью и
  раскладкой по белкам/жирам/углеводам (КБЖУ). Можно добавлять свои рецепты.
- **Подбор по продуктам.** На вкладке «Продукты → Продукты» отмечаются доступные ингредиенты,
  и приложение показывает подходящие рецепты, отсортированные по убыванию совпадений.
- **Фильтрация по тегам.** Вегетарианское, без глютена, кето, завтрак, ужин и т.д.
- **Сортировка и фильтрация по КБЖУ.** Режимы «Легко» (до 300 ккал), «Сытно» (300–600),
  «Все», а также сортировка по калориям и нутриентам.
- **Поиск по названию.**
- **Дневник питания.** Подсчёт съеденных калорий и БЖУ, цели рассчитываются по формуле
  Миффлина — Сан Жеора, трекер воды.
- **Список покупок и холодильник.** Ингредиенты из рецепта можно отправить в список покупок,
  а купленное — переместить в холодильник.
- **Пошаговое приготовление** со встроенным таймером, который сам распознаёт время из
  текста шага.

## Архитектура

Код разделён на статическую библиотеку логики `RecipeCore` и графическую часть на QML.
Тесты подключают ту же `RecipeCore`, поэтому проверяется именно тот код, из которого
собрана программа.

| Слой | Файлы | Назначение |
|------|-------|------------|
| Данные | `Recipe.h`, `RecipeRepository` | Структура рецепта и доступ к таблице SQLite |
| Логика | `RecipeService` | Подбор, фильтрация, расчёт КБЖУ, разбор шагов |
| Хранилище | `DatabaseManager` | Создание таблиц, начальное заполнение, избранное, покупки, холодильник |
| Представление | `RecipeModel` | Модель для QML (`QAbstractListModel`) |
| Ошибки | `RecipeException` | Единый тип исключений |

Все ошибки выбрасываются через исключения (`RecipeException`) и обрабатываются в `try/catch`.
Ручного управления памятью (`new`/`delete`) нет: используются объекты на стеке и механизм
владения через родителей в Qt.

## Установка и сборка

Требуется CMake ≥ 3.16, компилятор с поддержкой C++17 и установленный Qt 6
(модули Core, Gui, Qml, Quick, Sql, QuickControls2).

```bash
cmake -B build -S . -DCMAKE_PREFIX_PATH=/путь/к/Qt6
cmake --build build
```

Запуск:

```bash
./build/appRecipeWizard
```

> Изображения рецептов кладутся в `qml/images/` под именами `1.jpg … 55.jpg`. Если файла
> для рецепта нет, в интерфейсе показывается заглушка — на работу это не влияет.

### Настраиваемые пути

Абсолютных путей в коде нет, в качестве разделителя используется `/`. При запуске можно
указать произвольные пути к файлу базы данных и к папке с картинками:

```bash
./build/appRecipeWizard --db /home/user/my_recipes.db --images /home/user/recipe_photos
```

| Опция | Короткая | По умолчанию | Назначение |
|-------|----------|--------------|------------|
| `--db` | `-d` | `recipes.db` | путь к файлу SQLite |
| `--images` | `-i` | встроенные ресурсы | папка с фотографиями рецептов |

## Тесты

Тесты написаны с использованием doctest и запускаются через `ctest` из каталога сборки:

```bash
cmake --build build
cd build
ctest --output-on-failure
```

Покрыты положительными и отрицательными случаями: масштабирование ингредиентов под число
порций, очистка названия продукта, разбор шагов и таймера, расчёт нутриентов, добавление
и поиск рецептов в репозитории, ранжирование подбора и фильтрация по тегам.

## Документация

Документация к коду оформлена в формате Doxygen прямо в заголовочных файлах. Для сборки
HTML-версии в папку `docs/` нужно выполнить из корня проекта:

```bash
doxygen Doxyfile
```

Готовая документация откроется по адресу `docs/html/index.html`.

## Пример использования

1. Откройте вкладку **Продукты → Продукты** и добавьте то, что есть дома (например, «яйца», «сыр»).
2. Отметьте нужные продукты галочками и нажмите **«НАЙТИ РЕЦЕПТЫ»**.
3. Приложение покажет блюда, отсортированные по числу совпавших ингредиентов.
4. Откройте рецепт, при желании отправьте недостающие ингредиенты в список покупок и
   запустите пошаговое приготовление с таймером.
